# 穗学伴 · 自适应前端教学辅导 App 规划 V1.0

> 版本：v1.0 | 日期：2026-05-10 | 状态：规划稿
> 产品形态：响应式 Web App（自适应手机/iPad），非微信小程序
> 核心目标：替代家长为学生提供 AI 辅助教学辅导

---

## 1. 产品定位与范围

### 1.1 一句话定义

> 一个**自适应手机和 iPad** 的 Web 教学前端，学生独立完成练习、获取 AI 分步引导，家长在后台查看学习进度。

### 1.2 范围边界

| 包含（MVP） | 不包含（后继） |
|---|---|
| 学生端：出题 → 答题 → 判对错 → 分步引导 | 家教匹配系统 |
| 家长端：学习报告看板 | 教师/班级管理 |
| 自适应手机竖屏 + iPad 横竖屏 | 语音交互 |
| 基于已有题库出题（三年级435题 + 三年级下89题） | 记忆宫殿贴纸系统（作为独立功能） |
| 错题自动收集 | 视频/直播教学 |
| 知识点掌握度追踪 | 用户注册/登录系统（初期单机 Demo） |

### 1.3 与 PRD 的关系

PRD v0.3 原定微信小程序。考虑到开发效率和快速验证，本规划将 MVP 形态调整为 **Web App（PWA）**，原因：

- 开发速度快（单 HTML + CSS + JS，无需微信审核）
- 可先在手机/iPad 浏览器中打开验证，后续再打包微信小程序
- 题库数据已就绪（JSON 格式），可直接挂载

## 2. 用户角色与故事

### 2.1 学生（主要使用者）

**画像**：7-10 岁，三年级/四年级学生，注意力 15-25 分钟
**痛点**：家长没时间辅导，自己做题无人指导
**核心诉求**：做完题知道对错，不会的有人教

**典型故事**：
1. 小明打开穗学伴 → 选择"三年级下册" → 选"两位数乘两位数" → 做 10 道题 → 错了 3 道 → 系统给出分步引导帮助理解 → 错题自动收录
2. 小红连续打卡 5 天 → 看到自己的掌握度提升了 → 获得奖励

### 2.2 家长（监督者）

**画像**：30-45 岁，工作忙，关注孩子数学成绩
**痛点**：不知道孩子哪里不会，没时间一题一题看
**核心诉求**：快速了解薄弱点和进步

**典型故事**：
1. 家长打开 App → 看每日小结（做题数/正确率/用时）→ 看到"面积"章节掌握度 45% → 推送给孩子对应练习

## 4. 技术架构

### 4.1 架构决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 前端框架 | 纯 HTML + CSS + JS（单页应用） | MVP 阶段无需框架，减少依赖和构建步骤 |
| 数据存储 | localStorage（MVP）/ 后续可升级为 IndexedDB | 零后端，离线可用 |
| 题库格式 | JSON（已有） | 直接读取，无需转换 |
| 自适应方案 | CSS Media Queries + rem/vw 布局 | 天然适配手机竖屏和 iPad 横竖屏 |
| AI 分步引导 | 本地规则引擎（先写规则版，后续接入 LLM API） | MVP 阶段先跑通非 AI 版 |
| 部署方式 | 静态 HTML 文件，直接打开或轻量 http 服务 | 无需服务器，无需构建 |

### 4.3 数据流

```
用户选择章节/知识点
       ↓
quiz-engine 从题库 JSON 中筛选题目
       ↓
       ↓ 应用随机+优先级规则（优先出薄弱知识点）
       ↓
页面渲染题目 → 用户作答 → 判对错
       ↓             ↓
    答对        答错
       ↓             ↓
   → 更新掌握度     → 收录错题本
                    → 展示分步引导
                    → 出同类题补练
       ↓
完成总结 → 更新 localStorage → 刷新家长看板
```

## 5. 功能规格（MVP）

### 5.1 首页

- 年级选择（三下/四上/四下），默认四年级上册
- 今日推荐：系统推荐 2 个薄弱知识点作为当日目标
- 学习统计概览：总做题数、今日正确率、连续打卡天数
- 快速入口：开始练习 / 错题本 / 家长看板

### 5.2 章节选择

- 按学期展示章节列表
- 每章节显示掌握度进度条
- 已掌握（>80%）/ 学习中 / 未开始的视觉区分
- 支持按知识点展开/折叠

### 5.3 答题界面

| 元素 | 说明 |
|------|------|
| 题目区域 | 居中显示题目文本，数学公式用 Unicode 或 MathML |
| 题型适配 | 口算（输入框）、选择（4 选项按钮）、填空（输入框）、判断（对/错按钮）、应用题（输入框） |
| 计时器 | 可选开启（显示耗时） |
| 即时反馈 | 答对：绿色打勾 + 音效；答错：红色叉 + 显示正确答案 |
| 分步引导 | 应用题答错后展示解题思路（分步骤展开） |
| 导航 | 上一题/下一题，进度条显示当前在 N 题中的位置 |

### 5.4 答题结果

- 本组练习统计：总题数/正确数/正确率/用时
- 逐题回顾：可展开每道题的详情（题目、答案、你的回答、解析）
- 错题标记：红色高亮
- 一键重练错题
- 推荐下一步：推荐另一个相关知识点

### 5.5 错题本

- 按知识点分类展示
- 每条显示：题目、正确答案、你的错误答案、答错时间
- 一键重练（从当前错题中随机抽题）
- 清除已掌握的错题（连续答对 3 次同类题自动标记）

### 5.6 家长看板

- 知识点掌握度雷达图（使用 Canvas 绘制）
- 正确率趋势折线（最近 7 天/14 天）
- 薄弱知识点列表（掌握度 < 60%）
- 学习时长分布
- 年级切换（查看不同孩子的进度——预留）

## 6. 自适应设计规范

### 6.1 断点

| 设备 | 断点 | 布局 |
|------|------|------|
| 手机竖屏 | < 600px | 单列布局，全宽卡片 |
| 手机横屏 / 小平板 | 600px - 900px | 两列，侧边导航 + 主内容 |
| iPad 竖屏 | 768px - 834px | 两列，侧边栏固定 |
| iPad 横屏 / 桌面 | > 1024px | 宽版布局，更多信息密度 |

### 6.2 设计语言

- **配色**：金色/暖深色（与 PPT 对比 HTML 一致）
- **字体**：PingFang SC（iOS）/ Noto Sans SC（Android），大字号优先
- **按钮尺寸**：最小触控区域 44×44px（Apple HIG），儿童模式下 56×56px
- **间距**：核心间距 16px→24px（舒适阅读）
- **动画**：减少非必要动画，过渡用 200ms ease

### 6.3 无障碍

- 对比度 ≥ 4.5:1
- 支持字号放大（1.25x / 1.5x）
- 按钮和可交互元素有明确视觉反馈

## 7. 题库集成

### 7.1 已有数据

| 年级 | 题型 | 题量 | 文件路径 |
|------|------|:----:|----------|
| 三年级下册 | 口算/填空/选择/判断/笔算/应用 | 89 题 | `题库/三年级下册_题库.json` |
| 四年级上册 | 口算/填空/选择/应用 | 235 题 | `题库/上册_题库.json` |
| 四年级下册 | 口算/填空/选择/应用 | 200 题 | `题库/下册_题库.json` |
| **合计** | | **524 题** | |

### 7.2 题型映射

| JSON type | 前端交互组件 |
|-----------|-------------|
| `口算` | 数字输入框（自动聚焦，支持键盘回车） |
| `填空` | 文本输入框 |
| `选择` | 4 选项按钮（A/B/C/D 标记） |
| `判断` | 对/错两按钮 |
| `笔算` | 数字输入框（支持多行） |
| `应用` | 文本输入框 + 分步引导触发 |

### 7.3 出题策略

1. 优先出薄弱知识点（掌握度 < 60%）的题目
2. 同一知识点内随机选题，避免重复
3. 每轮练习默认 10 题，支持自定义
4. 相邻题目知识点尽量不同（提高覆盖面）

## 9. 风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|------|:----:|:----:|------|
| Web App 在微信内打开体验差 | 中 | 中 | MVP 优先在浏览器/Safari 使用；后续打包小程序 |
| 儿童操作不友好（误触、读不懂题） | 中 | 高 | 大按钮、大字号、语音朗读（P1）；早期找 1-2 个孩子做可用性测试 |
| 分步引导（规则版）覆盖不全 | 高 | 中 | 先覆盖主流应用题题型；P1 升级 LLM API |
| 题库数量不够支撑每日练习 | 中 | 中 | 当前 524 题够 MVP 验证；继续补充至 1000+ |
| 家长看板数据不直观 | 低 | 中 | 原型阶段收集反馈迭代 |
