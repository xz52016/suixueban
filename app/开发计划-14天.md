## W1：核心骨架 + 答题功能

### Day 1 — 项目骨架

**目标**：文件结构就位，CSS 主题渲染正常，路由跑通

- [ ] 创建 `app/` 目录下所有文件夹
- [ ] 配置 CSS 变量（主题色、间距、字体）→ `css/theme.css`
- [ ] 实现响应式布局基础 → `css/layout.css`（手机 + iPad 双端骨架）
- [ ] 实现 hash 路由 → `js/router.js`
- [ ] 编写 `index.html` 入口 + 路由容器
- [ ] 验证：浏览器打开看到穗学伴首页骨架

**关键决定**：
- 主题色：金色 #C9A84C / 深底 #1a1a1a / 暖白 #f5f0e0
- 路由模式：`#home` → 首页，`#chapter` → 章节选择，`#quiz` → 答题
- 数据绑定：先用 string-based template（MVP 不引入框架）

### Day 2 — 首页 + 章节选择

**目标**：完整首页，章节选择页面可从题库 JSON 读取数据

- [ ] 实现首页页面模板 → `pages/home.html`
- [ ] 从 `data/` 读取题库 JSON（fetch or inline）
- [ ] 实现章节选择页面 → `pages/chapter-select.html`
- [ ] 章节列表中显示每章掌握度进度条
- [ ] 绑定点击事件：选章节 → 跳转答题页
- [ ] 验证：选三年级下册→显示9章→点第4章→跳转答题页

### Day 3 — 出题引擎

**目标**：核心算法——从题库中筛选+随机选题

- [ ] 实现 `js/quiz-engine.js`
  - `loadQuestions(grade, chapter)` — 从 JSON 加载题目
  - `pickQuestions(questions, count, excludeWrongIds)` — 随机选 N 题
  - `shuffle(array)` — Fisher-Yates 洗牌
- [ ] 持题型过滤（可选：只做应用题 / 只做口算）
- [ ] 出题策略：优先薄弱知识点（后续与 store 联动）
- [ ] 单元测试（手动 console.log 验证出题逻辑）

### Day 4 — 答题界面核心

**目标**：渲染题目，接收用户输入，判对错

- [ ] 实现 `pages/quiz.html` 模板
- [ ] 渲染组件：
  - 口算 → 数字输入框 + 回车提交
  - 填空 → 文本输入框
  - 选择 → A/B/C/D 四个按钮
  - 判断 → 对✅ / 错❌ 按钮
  - 笔算 → 数字输入框
- [ ] 即时判对错逻辑（精确匹配 + 去空格 + 忽略标点）
- [ ] 进度条（`当前题数/总题数`）
- [ ] 答案反馈动画（绿色勾 / 红色叉）

### Day 5 — 答题结果页面 + 数据持久化

**目标**：一组练习完成后看到结果，数据存到 localStorage

- [ ] 实现 `js/store.js`
  - `saveResult(grade, chapter, answers[])` — 保存答题记录
  - `getProgress(grade, chapter)` — 获取掌握度
  - `getWrongQuestions(grade)` — 获取错题列表
  - `getStreak()` — 连续打卡天数
- [ ] 实现 `pages/result.html` 结果展示页面
  - 总题数 / 正确数 / 正确率
  - 逐题回顾（正确/错误标记）
  - 一键重练错题 / 返回首页
- [ ] 验证：做完10题→看结果→刷新页面→数据还在

## W2：错题本 + 看板 + 分步引导

### Day 6 — 错题本

**目标**：错题收集 + 按知识点分类 + 一键重练

- [ ] 实现 `pages/wrong-book.html`
- [ ] 按知识点分组展示错题
- [ ] 每条显示：题目 / 你的答案（红） / 正确答案（绿） / 答错时间
- [ ] 一键重练：从当前错题组抽 N 题
- [ ] 自动移除：连续答对 3 次的错题标记为已掌握

### Day 7 — 家长看板：数据层

**目标**：掌握度计算逻辑就绪，可输出图表所需数据

- [ ] 实现 `js/progress.js`
  - `calculateMastery(grade)` → 返回每章节掌握度（0-100%）
  - `getTrend(grade, days)` → 返回每日正确率数组
  - `getWeakPoints(grade)` → 掌握度 < 60% 的知识点
  - `getTodayStats()` → 今日做题数/正确率/用时
- [ ] 存储每日快照（localStorage 中按日期索引）

### Day 8 — 家长看板：图表

**目标**：雷达图 + 折线图渲染

- [ ] 实现 `pages/dashboard.html`
- [ ] Canvas 绘制知识点掌握度雷达图（5-10 边形）
  - 每个顶点 = 一个章节
  - 填充区域 = 掌握度数值
- [ ] Canvas 绘制正确率趋势折线图
  - X 轴：最近 7 天日期
  - Y 轴：正确率 0-100%
- [ ] 薄弱知识点列表（带进度条 + 跳转练习）
- [ ] 今日学习概况卡片

### Day 9 — 分步引导（规则版）

**目标**：应用题答错后展示解题思路

- [ ] 实现 `js/step-guide.js`
  - 为常见题型注册规则：
    - 面积类：`长×宽=面积` / `边长×边长=面积`
    - 除法类：`总数÷份数=每份数` / `总数÷每份数=份数`
    - 乘法类：`单价×数量=总价`
    - 时间计算：`结束时间-开始时间=经过时间`
    - 搭配类：`A数×B数=总搭配数`
  - `getGuide(questionType, questionText)` → 返回步骤数组
  - 分步展开UI：第一步 → 用户点"下一步" → 第二步...
- [ ] 验证：选一个面积应用题 → 故意答错 → 看到分步引导

## 技术清单

| 技术 | 用途 | 理由 |
|------|------|------|
| 纯 HTML/CSS/JS | 前端框架 | 零依赖，零构建，直接预览 |
| CSS Media Queries | 自适应 | 原生支持，全面覆盖手机/iPad |
| Canvas API | 图表绘制 | 无需图表库，轻量 |
| localStorage | 数据持久化 | 简单可靠、离线可用 |
| Hash-based Router | 页面路由 | 无需后端，兼容性好 |
| Web Audio API | 音效 | 无需音频文件，零额外加载 |
